Tasks

Task parameters:

  • Task name: Custom user-definined name for identification of the task, can contain spaces and special characters. Name must be unique.
  • Interval (between cycles): If task is repeated more then once, it will be executed after this time in milliseconds.
  • Pool size: If the pool is bigger than 1, more parallel connection to the database will be created. SQL sets will be still executed in the defined order, but if the task did no finished before the next repeat is executed, the next repeat will run parallel on another connection.
  • Repeat count: Task can be repeated multiple times or in case of value -1 the task will run until manual stop.
  • SQL sets: Every task can have multiple SQL sets to execute. Sets are executed in the given order.
  • Databases: Task run on multiple databases simultaneously executing the exact same SQL sets on all databases in the same time.
